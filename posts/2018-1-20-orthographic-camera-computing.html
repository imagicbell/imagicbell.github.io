Restrict Object In Specified Screen Area Using Orthographic Camera

January 20, 2018
3min read

I’ve encountered a problem to adjust the position of orthographic camera for restricting 3D objects in the specified rectangle area in the screen. I was sure of the possibility, but it did take me some effort to accomplish it. Here is what I did.

Prerequisite

  1. Given the objects, whose center is at Vector.zero.

  2. Given the rect, with the proportional value of the display area relative to the screen, e.g. (0.1, 0.2, 0.5, 0.6) means the display area starts from (Screen.width*0.1, Screen.height*0.2), with size of (Screen.width*0.5, Screen.hegith*0.6).

    However, the actual display rect on screen should take the aspect ratio of the objects' bounds into consideration, so what we actually do is to limit the objects inside the specified area. We will provide it it details below.

  3. Given the pitch angle θ\thetaθ of the camera.

  4. Given the distance from the center of objects to the center of the camera.

See the picture below:

1. Get Bounding Box bound

This was done by accumulating all the mesh bounds of the objects. The center is fixed at Vector3.zero, as we designed to fix the objects in the scene and calculate the position of the camera to adjust the view.

2. Decide Orthographic Size

Based on the Unity's doc about orthographic size:

It is half the size of the vertical viewing volume. The horizontal size of the viewing volume depends on the aspect ratio.

So let's calucate the height of the view volume.

Fortunately, the calucation doesn't need spatial transformation. All is done in global space, as we use orthographic camera and the proportianl value.

According to the picture above, we can get α\alphaα by the bounding box calulated from step 1.

α=arctan(bound.heightbound.depth)\alpha = arctan(\frac{bound.height} {bound.depth})α=arctan(bound.depthbound.height​)

Then d:

d=bound.heightsinαd = \frac {bound.height} {sin\alpha}d=sinαbound.height​

Then the height of the object on screen, we call it view.height:

view.height=d×sin(α+θ)=bound.heightsinα×sin(α+θ)view.height = d \times sin(\alpha + \theta) = \frac {bound.height}{sin\alpha} \times sin(\alpha + \theta)view.height=d×sin(α+θ)=sinαbound.height​×sin(α+θ)

So the height of camera view volume is:

camera.height=view.heightrect.height=bound.heightrect.height×sin(α+θ)sinαcamera.height = \frac{view.height} {rect.height} = \frac {bound.height} {rect.height} \times \frac{sin(\alpha + \theta)} {sin\alpha}camera.height=rect.heightview.height​=rect.heightbound.height​×sinαsin(α+θ)​

Remember in Prerequisite 2, the rect is defined with the proportional value relative to the screen.

However, that's not enough. We should ensure the aspect ratio, and the bounding of object's view not exceeding the view volume. So we calculate the width of camera view volume:

camera.width=camera.height×camera.aspectcamera.width = camera.height \times camera.aspectcamera.width=camera.height×camera.aspect camera.width.needed=bound.widthrect.widthcamera.width.needed = \frac{bound.width}{rect.width}camera.width.needed=rect.widthbound.width​

if camera.width > camera.width.needed, it means that actually more width is given than that the object needs, so we should change the actual proportional value of the object's width on screen:

rect.width=rect.width×camera.width.neededcamera.widthrect.width = rect.width \times \frac {camera.width.needed}{camera.width}rect.width=rect.width×camera.widthcamera.width.needed​ rect.x=0.5+(rect.x−0.5)×camera.width.neededcamera.widthrect.x = 0.5 + (rect.x - 0.5)\times\frac{camera.width.needed}{camera.width}rect.x=0.5+(rect.x−0.5)×camera.widthcamera.width.needed​

if camera.width < camera.width.needed, it means that the actual width of object's view exceeds the camera's view volume. So we need to extend the view volume:

camera.width=camera.width.neededcamera.width = camera.width.neededcamera.width=camera.width.needed

Let's save the old camera.height before computing the new camera.height:

camera.height.needed=camera.heightcamera.height.needed = camera.heightcamera.height.needed=camera.height camera.height=camera.widthcamera.aspectcamera.height = \frac{camera.width}{camera.aspect}camera.height=camera.aspectcamera.width​

Now that camera.height > camera.height.needed, we need to re-compute the actual proportional value of the object's height on screen, same as width above:

rect.height=rect.height×camera.height.neededcamera.heightrect.height = rect.height \times \frac {camera.height.needed}{camera.height}rect.height=rect.height×camera.heightcamera.height.needed​ rect.y=0.5+(rect.y−0.5)×camera.height.neededcamera.heightrect.y = 0.5 + (rect.y - 0.5)\times\frac{camera.height.needed}{camera.height}rect.y=0.5+(rect.y−0.5)×camera.heightcamera.height.needed​

then

camera.orthographicSize=camera.height×0.5camera.orthographicSize = camera.height \times 0.5camera.orthographicSize=camera.height×0.5

3. Compute Position

Keep the picture above and the parameters offered in mind, and then continue:

d1=camera.height×(rect.y+rect.height−0.5)d_1 = camera.height \times (rect.y+rect.height-0.5)d1​=camera.height×(rect.y+rect.height−0.5) deltaX=d1×sinθdeltaX = d_1 \times sin\thetadeltaX=d1​×sinθ deltaY=d1×cosθdeltaY = d_1 \times cos\thetadeltaY=d1​×cosθ y1=(distance−0.5×bound.depth+deltaX)×tanθy_1 = (distance - 0.5 \times bound.depth + deltaX) \times tan\thetay1​=(distance−0.5×bound.depth+deltaX)×tanθ camera.y=y1+deltaYcamera.y = y_1 + deltaYcamera.y=y1​+deltaY camera.x=−distancecamera.x = -distancecamera.x=−distance

Then for camera.z:

camera.z=−(rect.x+0.5×rect.width−0.5)×camera.widthcamera.z = - (rect.x + 0.5 \times rect.width - 0.5) \times camera.widthcamera.z=−(rect.x+0.5×rect.width−0.5)×camera.width

Finally:

camera.position = new Vector3(camera.x, camera.y, camera.z)

Yeah, that's it!!! Happy?
